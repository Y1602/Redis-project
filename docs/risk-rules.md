# 风险规则设计

## 1. 风险分数累计周期
风险分数按天累计计算
key示例：
    risk:rank:2026-04-27
含义：
    记录 2026-04-27 当天所有 IP 的风险分。

## 2. 基础风险规则
| 行为          | 分数 |
| ----------- | -： |
| 访问 `/login` | +2 |
| 访问 `/admin` | +5 |
| 状态码 `200`  | +0 |
| 状态码 `401`   | +3 |
| 状态码 `403`   | +3 |
| 状态码 `500`   | +1 |

## 3. 风险计算逻辑
根据每次请求的path和status计算本次风险数值
示例1：GET /login 200
      风险分：+2
示例2：GET /admin 403
      风险分：+8

## 4. 风险分写入方式
使用ZSet记录风险排行：ZINCRBY risk:rank:{date} {score} {ip}
示例：
     ZINCRBY risk:rank:2026-04-27 8 10.0.0.1

## 5. 封禁规则
当某个IP当天累计风险分达到阈值时，触发临时封禁。
第一阶段阈值：
      风险分 >= 20
封禁时间：
      60秒
redis操作：
      SET ban:ip:{ip} 1 EX 60

## 6. 封禁判断逻辑：
处理请求前，先检查：GET ban:ip:{ip}。如果存在则拒绝处理，输出blocked；如果不存在则继续处理访问记录和风险计算。

## 7. 第一阶段规则说明
第一阶段只做简单规则，不引入复杂算法。
